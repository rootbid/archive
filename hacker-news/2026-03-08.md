# Hacker News Daily Summary - 2026-03-08

**[Plasma Bigscreen – 10-foot interface for KDE plasma](https://plasma-bigscreen.org)**
- **Summary:** A user-friendly, open-source, Wayland-based desktop environment designed for HTPCs and SBCs like the Raspberry Pi. It offers a 10-foot UI optimized for TVs and projectors, allowing control via CEC remotes, gamepads, or phones.
- **Discussion:** Users are comparing it to Kodi and LibreELEC, noting its superior flexibility for general computing and web browsing. Major points of debate include the difficulty of supporting proprietary streaming services (DRM) and the trade-off between a dedicated media player vs. a full desktop.

**[Uploading Pirated Books via BitTorrent Qualifies as Fair Use, Meta Argues](https://torrentfreak.com/uploading-pirated-books-via-bittorrent-qualifies-as-fair-use-meta/)**
- **Summary:** Meta is defending its use of pirated books for AI training (Books3) by claiming that the upload process inherent to BitTorrent qualifies as "fair use." The argument hinges on the fact that uploading is a technical necessity for downloading the datasets required for AI development.
- **Discussion:** HN users are debating the legal implications for individual users of torrenting, many noting the irony of a massive corporation using "fair use" as a shield. Technical discussions focus on the "seeding" requirement of the BitTorrent protocol and whether courts will accept "technical necessity" as a valid defense.

**[UUID package coming to Go standard library](https://github.com/golang/go/issues/62026)**
- **Summary:** A proposal for adding a `crypto/uuid` package to the Go standard library has been accepted, supporting generation and parsing of UUIDs per RFC 4122 and RFC 9562 (version 7).
- **Discussion:** The Go community is largely supportive as it reduces dependency on external libraries like `google/uuid`. Significant debate exists around whether to support non-standard UUID formats and the performance implications of cryptographic RNG sources.
- **HN Thread:** [https://news.ycombinator.com/item?id=47283665](https://news.ycombinator.com/item?id=47283665)

**[Ki Editor - an editor that operates on the AST](https://ki-editor.org/)**
- **Summary:** Ki Editor is a structured code editor that operates directly on the Abstract Syntax Tree (AST) rather than raw text. It allows for advanced navigation and manipulation of syntax nodes, making refactoring-heavy workflows more efficient.
- **Discussion:** HN users are drawing parallels to JetBrains' "structural selection" and older tools like Lisp machines. There is a healthy debate over "text-first" vs. "structure-first" philosophies, especially regarding the difficulty of handling syntactically invalid code during editing.
- **HN Thread:** [https://news.ycombinator.com/item?id=47286311](https://news.ycombinator.com/item?id=47286311)
