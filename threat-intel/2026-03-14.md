# Sentinel Threat Intel: 2026-03-14
## High-Impact Security Threats Brief

### 1. Google Chrome V8 and Skia Zero-Days (Exploited In-The-Wild)
**CVEs:** CVE-2026-3909, CVE-2026-3910
**CVSS:** 8.8 (High/Critical)
**Description:** Google Chrome is under active attack leveraging two zero-day vulnerabilities in the V8 JavaScript/WebAssembly engine (CVE-2026-3910) and Skia. They allow arbitrary remote code execution via a crafted HTML page, potentially bypassing sandbox protections.
**Target:** Chrome browsers, Electron/Chromium-based applications.
**IoCs/Mitigation:** 
- Immediately update Google Chrome and Chromium-based browsers to the latest patched version.
- Ensure automated patching is enforced across all endpoints.
- Monitor for anomalous child processes spawned by browser executables.

### 2. n8n Critical Remote Code Execution (Zero-Day at Discovery)
**CVE:** CVE-2026-25049
**CVSS:** 9.4 (Critical)
**Description:** A critical remote code execution vulnerability was discovered in n8n, a widely used workflow automation tool. This allows unauthenticated or low-privileged attackers to execute arbitrary commands on the underlying host, directly impacting automated infrastructure and cloud deployments.
**Target:** Automated systems, CI/CD pipelines, Cloud infrastructure running n8n.
**IoCs/Mitigation:**
- Upgrade n8n instances immediately to the patched release.
- Restrict network access to the n8n web interface to trusted IPs/VPNs.
- Monitor execution of suspicious shell commands originating from the n8n application process.

### 3. Veeam Backup Server Pre-Auth RCE Flaws
**CVEs:** CVE-2026-21666, CVE-2026-21667, CVE-2026-21669
**CVSS:** 9.8 (Critical)
**Description:** Three critical remote code execution vulnerabilities expose Veeam backup servers to low-complexity attacks. Low-privileged domain users can exploit these flaws to execute remote code on the backup servers, posing a severe threat to disaster recovery infrastructure and opening the door for ransomware deployment.
**Target:** Cloud infrastructure and enterprise backup systems.
**IoCs/Mitigation:**
- Apply the latest Veeam security patches immediately.
- Implement network segmentation to isolate backup servers from general user networks.
- Enforce strict Least Privilege Access for domain users.

### 4. Microsoft AI Research Repository RCE
**CVE:** CVE-2026-23654
**CVSS:** High
**Description:** A high-severity remote code execution vulnerability affects the Microsoft AI research repository `microsoft/zero-shot-scfoundation`. This targets developers and automated pipelines utilizing this AI research framework, potentially leading to supply-chain attacks or compromised research environments.
**Target:** AI systems, developer workstations, ML pipelines.
**IoCs/Mitigation:**
- Update dependencies and pull the latest patched version of the repository.
- Run AI research frameworks in isolated, ephemeral sandbox environments (e.g., restricted containers).
