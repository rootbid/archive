# Sentinel Threat Intelligence Brief: 2026-02-28

## Executive Summary
This report highlights three critical high-impact vulnerabilities actively shaping the threat landscape as of late February 2026. Immediate attention is required for **Google Chrome/Chromium** (Zero-Day under active attack), **BeyondTrust Remote Support** (Critical Pre-Auth RCE with public PoC), and **n8n Workflow Automation** (RCE targeting AI/automation pipelines).

## Critical Threats

### 1. Google Chrome / Chromium Zero-Day (CVE-2026-2441)
*   **Type:** CSS Use-After-Free leading to Remote Code Execution (RCE).
*   **Severity:** **High** (Active Exploitation Confirmed).
*   **Impact:** Allows attackers to execute arbitrary code within the browser sandbox via malicious web content. Successful exploitation can lead to full system compromise if chained with a sandbox escape.
*   **Affected Systems:** Google Chrome and all Chromium-based browsers (Edge, Brave, Opera).
*   **Status:** **Active In-the-Wild Exploitation**.
*   **Mitigation:**
    *   **Immediate Patching:** Update Chrome to the latest stable release immediately.
    *   **Endpoint Defense:** Ensure EDR solutions are active and monitoring for abnormal browser child processes.

### 2. BeyondTrust Remote Support & PRA RCE (CVE-2026-1731)
*   **Type:** Unauthenticated Remote Code Execution (Command Injection).
*   **Severity:** **Critical** (CVSS 9.9).
*   **Impact:** Allows unauthenticated remote attackers to execute arbitrary operating system commands as the site user. This provides full control over the appliance and potentially the managed infrastructure.
*   **Affected Systems:** BeyondTrust Remote Support (RS) and Privileged Remote Access (PRA) - unpatched versions.
*   **Status:** **Public PoC Available** (GitHub, Feb 10) & **Active Reconnaissance/Exploitation**.
*   **Mitigation:**
    *   **Patch Now:** Apply vendor patches immediately.
    *   **Network Isolation:** Restrict access to the management interface to trusted IPs only; ensure it is not exposed to the public internet if possible.
    *   **IoC Monitoring:** Watch for anomalous traffic to the `thin-scc-wrapper` component.

### 3. n8n Workflow Automation RCE (CVE-2026-25049)
*   **Type:** Sandbox Escape / Expression Injection leading to RCE.
*   **Severity:** **Critical** (CVSS 9.4).
*   **Impact:** Authenticated users (or attackers compromising a low-priv account) can bypass sandbox restrictions via crafted expressions to execute arbitrary system commands on the host server.
*   **Target:** AI/Automation pipelines and internal workflow orchestration.
*   **Affected Systems:** n8n versions prior to 1.123.17 and 2.5.2.
*   **Status:** **PoC Available**; targeted by attackers aiming to hijack automation infrastructure.
*   **Mitigation:**
    *   **Update:** Upgrade to n8n version 1.123.17, 2.5.2, or later.
    *   **Access Control:** Strictly limit user permissions for creating/modifying workflows.
    *   **Network Segmentation:** Isolate the n8n host from sensitive internal networks.
