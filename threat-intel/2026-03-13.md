# Sentinel Threat Intelligence: 2026-03-13

## Executive Summary
This brief highlights critical security threats observed in the wild targeting widely used infrastructure, focusing on remote code execution (RCE) and high exploitation potential.

## 1. Google Chrome Zero-Day RCE (CVE-2026-2441)
**Component:** Google Chrome / Chromium Engine
**Threat Level:** CRITICAL (Actively Exploited In-The-Wild)
**Description:** A high-severity use-after-free (UAF) vulnerability in the CSS component allows remote attackers to execute arbitrary code within the Chrome sandbox via maliciously crafted web pages.
**Impact:** Allows full compromise of the browser session. If paired with a sandbox escape, it leads to full system compromise.
**IoCs/Detection:** 
- Monitor for anomalous browser crashes or unusual sub-process execution originating from `chrome.exe` / `chrome`.
**Mitigation:** 
- Immediately update Google Chrome and all Chromium-based browsers (Edge, Brave, Opera) to the latest patched versions.
- Restrict browser access to untrusted domains where feasible.

## 2. Node.js `node-tar` Arbitrary File Overwrite / RCE (CVE-2026-29786)
**Component:** `node-tar` package (Node.js ecosystem)
**Threat Level:** HIGH (High Exploitation Potential)
**Description:** A link following vulnerability during extraction allows an attacker to overwrite arbitrary files on the system (e.g., `~/.ssh/authorized_keys`, `/etc/cron.d/`, or application code) if the extraction process has write permissions. 
**Impact:** Can be trivially escalated to Remote Code Execution (RCE) by overwriting critical system scripts or configuration files, especially dangerous for automated pipelines and cloud infrastructure processing user-supplied archives.
**IoCs/Detection:**
- Monitor for suspicious file modifications outside the intended extraction directory.
- Audit logs for modifications to SSH keys, cron jobs, or global `node_modules`.
**Mitigation:**
- Update the `node-tar` package to the latest patched version across all Node.js projects.
- Execute archive extraction with the principle of least privilege (e.g., non-root user, restricted file system access).
